# TSE 2026 Replication Package

This repository contains the replication package for the experimental evaluation of the IEEE TSE 2026 article. The package includes the necessary scripts, datasets, and Docker configurations to reproduce the static analysis experiments reported in the paper.

## Repository Structure

- `docker/`: Contains all the logic and environment configuration required to run the experiments. This is the main directory you will interact with. It uses a custom Docker image to ensure reproducibility.
- `tse2026/`: Contains intermediate data or output generated by previous runs (if applicable).
- `split_large_files.sh` & `join_large_files.sh`: Utility scripts to handle large files (e.g., splitting large CSVs or archives for Git storage and rejoining them later).

## How to Reproduce the Experiments

All executions are managed through Docker containers to isolate the environment and dependencies (Java, Maven, Python, etc.).

1. Navigate to the `docker/` directory:
   ```bash
   cd docker/
   ```

2. Follow the detailed instructions inside the `docker/README.md` file to build the Docker image and execute the experiment orchestrator (`run_experiment.sh`).

### Quick Start

To execute the entire replication pipeline with the default configuration (10 containers processing both MDS and RDS datasets):

```bash
cd docker/
bash run_experiment.sh 2>&1 | tee experiment.log
```

*(Note: execution may take several hours depending on your hardware limits)*

## Results

Once the execution finishes, the results will be automatically collected from the containers and stored in the `docker/experiment_out_<timestamp>` (or configured) folder. Inside this folder, you will find:
- Statistical summary CSVs.
- Detailed execution logs.
- Generated charts/PDFs representing the results discussed in the article.

## Requirements

- **Docker:** Ensure the Docker daemon is running and has enough resources allocated (e.g., at least 4GB of RAM per container is recommended).
- **Bash:** Required to run the orchestrator scripts (can be used on Linux, macOS, or Windows via WSL/Git Bash).
